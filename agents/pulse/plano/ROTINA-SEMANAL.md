## Modelo de Sprint

- Ciclo semanal (segunda a domingo)
- Um sprint por area, sincronizados
- Cada issue com dono definido — sem task sem assignee
- Backlog sempre priorizado

---

## Rotina Diaria — Check-in das 9h

- Roda de **segunda a sexta** via cron
- Pulse puxa dados do task manager e envia **individualmente** para cada membro (DM separado)

### O que aparece no check-in

- 🚨 **Atrasados** — issues que passaram do prazo
- 📋 **Planejado para hoje** — issues com data de inicio = hoje
- ⏳ **Proximos vencimentos** — issues com prazo chegando
- 🟡 **Proximo inicio** — issues com data de inicio em breve
- Issues sem prazo **nao aparecem**

### Cobrancas automaticas

- Issue sem atualizacao por **2 dias** → Pulse cobra o responsavel direto no chat (DM)

---

## Reuniao Semanal de Planejamento

- Equipe se reune toda **segunda as 10h**
- Pulse entra no modo interativo: cria issues em tempo real

---

## Relatorio Semanal — Sexta-feira

- Pulse gera o relatorio de fechamento toda sexta 17h
- Enviado no chat para a equipe
- Salvo em `plano/relatorios/YYYY-MM-DD.md`

---

## Regras do backlog

**Sem OK do decisor:** task operacional, nao cabe no sprint, ideia nova
**Com OK do decisor:** feature nova, compromisso com cliente, muda prazo externo
